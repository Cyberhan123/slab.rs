你需要实现slab-file-search, 增强slab-file-watcher和slab-file-system
slab-file-search
功能：模糊文件搜索引擎，类似 fzf/telescope。

使用 nucleo 进行高性能模糊匹配
使用 ignore crate 遍历目录，支持 .gitignore 规则
采用两个后台线程：walker 线程（遍历目录）和 matcher 线程（执行模糊匹配）
支持 FileSearchSession（流式、可实时更新 query）和 run（一次性同步查询）两种使用方式
对外暴露 FileMatch、FileSearchSnapshot、SessionReporter 等类型

slab-file-watcher
功能：多订阅者文件系统变更监听器。

基于 notify crate 封装，支持跨平台 OS 原生文件监听
支持多个订阅者（FileWatcherSubscriber），每个订阅者独立接收变更通知
通过引用计数管理 OS 层的实际 watch 注册，避免重复监听同一路径
提供 ThrottledWatchReceiver 对事件进行节流（debounce）
支持不存在的路径：通过监听祖先目录，等路径被创建后自动升级 watch
处理符号链接、路径规范化（/var/... vs /private/var/...）等边界情况

slab-file-system
功能：抽象的文件系统访问 trait，用于本地/远端环境的统一接口。

定义 ExecutorFileSystem async trait，包含 read_file、write_file、create_directory、get_metadata、read_directory、remove、copy 等操作
提供 FileSystemSandboxContext，封装沙箱权限上下文（sandbox policy、cwd 等）
纯接口 crate，由上层 crate提供具体实现

下面是三个crate的建议依赖你可以自行调整：

slab-file-search 中的依赖为:
[dependencies]
anyhow = { workspace = true }
clap = { workspace = true, features = ["derive"] }
crossbeam-channel = { workspace = true }
ignore = { workspace = true }
nucleo = { workspace = true }
serde = { workspace = true, features = ["derive"] }
serde_json = { workspace = true }
tokio = { workspace = true, features = ["full"] }

[dev-dependencies]
pretty_assertions = { workspace = true }
tempfile = { workspace = true }


slab-file-system 中的依赖为:
[dependencies]
async-trait = { workspace = true }
serde = { workspace = true, features = ["derive"] }


slab-file-watcher 中的依赖为:
[dependencies]
notify = { workspace = true }
tokio = { workspace = true, features = ["macros", "rt", "sync", "time"] }
tracing = { workspace = true, features = ["log"] }

[dev-dependencies]
pretty_assertions = { workspace = true }
tempfile = { workspace = true }

通过这个文件系统的实现，我们的slab-app 中 workspace 的文件读取逻辑变换为：
slab-app->slab-file-system + slab-file-watcher+slab-file-search

需要确认slab-app 会不会过slab-app-core 如果过则调用逻辑为：
slab-app->slab-app-core->slab-file-system + slab-file-watcher+slab-file-search
