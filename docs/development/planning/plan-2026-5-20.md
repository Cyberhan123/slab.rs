McpClient结构体：连接到 MCP 服务器（stdio 子进程或 SSE/HTTP 端点）

JSON-RPC 2.0 消息帧：McpRequest，，McpResponseMcpNotification

调用方式：call_tool(name, args)→ 返回McpToolResult（内容数组）

生命周期：initialize()握手（协议协商）ping()、优雅关闭

自动将远程 MCP 工具注册ToolRouter为McpProxyTool适配器

McpServerToolRouter结构体：公开已注册为 MCP 服务器的slab。

传输方式：stdio（从标准输入读取，写入标准输出，兼容npx @modelcontextprotocol/inspector）

crates/slab-shell-command（新箱子）

提取并扩展slab-agent-tools/src/shell.rs成生产级独立包装箱。



ShellExecutor结构体：封装执行策略、工作区根目录、可选SandboxDriver

ShellCommand结构体：{ command: String, timeout_secs: u64, env: HashMap<String,String> }

ShellOutput结构体：{ stdout, stderr, exit_code, timed_out }

CommandSafetyChecker：扩展了来自 codex 的危险命令启发式规则（不仅限于 rm/dd/pipe 模式，还包括：fork 炸弹、内核崩溃触发器、引导加载程序写入、mkfs.*设备上的命令等chmod -R 777 /） 。

ExecPolicyChecker：SandboxPolicy根据命令咨询决定批准/需要批准/拒绝

集成方式：当注入slab-sandboxing一个参数时，将所有子进程的执行路由到该参数，而不是直接执行。SandboxDrivertokio::process::Command

PTY 支持：用于utils/pty交互式命令的方法

信号转发：将 Ctrl-C 传播到子进程组

输出截断：将标准输出/标准错误输出限制为可配置的字节数限制（默认为 100KB），并显示截断通知

更新slab-agent-tools/src/shell.rs委托给ShellExecutor

新增工具

apply_patch工具（由以下机构支持slab-file-system）：



接受统一差异补丁字符串，并将其应用到工作区。

返回{ applied_files: Vec<String>, result: "ok" | "error", error_message: Option<String> }


git_status工具（由以下机构支持slab-git）：



返回工作区的当前 Git 状态


git_diff工具（由以下机构支持slab-git）：



返回已暂存或未暂存的差异


git_commit工具（由以下机构支持slab-git）：



暂存所有更改并使用提供的消息提交。


mcp_call工具（由以下机构支持slab-mcp）：



将工具调用代理到指定的外部 MCP 服务器

模式：{ server_name, tool_name, arguments }
