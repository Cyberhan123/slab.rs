你需要调整slab-mcp 实现
将client 的核心实现放在 slab-mcp-client 里
将 server 实现放在bin/slab-mcp-server 里

调用关系：外部 MCP Server（第三方工具）
        ↑  连接/调用
  slab-mcp-client        ← 底层：单连接 MCP 客户端 
        ↑  使用
    slab-mcp              ← 中层：多 server 管理、工具聚合、权限、认证 
        ↑  使用
  slab app-core          ← slab-app-core 主体逻辑 

外部 AI 客户端（Claude Desktop 等）
        ↓  JSON-RPC over stdio
 slab-mcp-server (bin)   ← slab 作为 MCP server 对外提供能力
slab-mcp-client 保持纯粹的通信层，方便测试和替换底层传输 
slab-mcp 集中所有业务逻辑（auth、sandbox、tools cache），避免散落各处
slab-mcp-server 是独立进程入口点，职责单一，不会污染库代码
